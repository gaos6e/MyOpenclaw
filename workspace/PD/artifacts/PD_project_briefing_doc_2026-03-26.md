# 基于表型驱动的扰动靶标预测技术简报

本报告旨在总结基于表型驱动的靶标预测研究背景、核心深度学习模型框架的构建思路、近期模型调整与优化的实验记录，以及化学扰动测试的初步结果。

## 1. 执行摘要 (Executive Summary)

当前药物研发正经历从“靶标驱动”向“表型驱动”的复兴转变。研究表明，1999至2008年间FDA批准的许多首创药物（First-in-class）是在没有预设靶点假设的情况下通过表型数据发现的。本项目核心在于构建一个基于单细胞组学大模型的创新深度学习框架，利用外界扰动在转录组表达谱中留下的特异性“指纹”，直接逆向预测扰动靶标（基因集或化合物）。通过结合单细胞基础大模型嵌入与图神经网络（GCN、GAT、Transformer），该框架旨在克服传统间接预测方法计算繁琐、泛化性差及搜索空间受限等瓶颈。近期实验重点优化了数据划分、正则化技术应用及差异表达（Fold Change）计算机制，并在化学扰动数据集上取得了初步验证结果。

## 2. 核心主题深度分析

### 2.1 表型驱动预测的策略演进
研究背景指出，扰动靶标预测存在两种主要路径：
*   **间接预测：** 依赖基于基线表达谱加扰动条件的模型（如GEARS、scGPT、TranSiGen）先模拟响应，再与实际表达谱比对。此类方法在面对组合扰动时，候选集数量随变量呈指数级增长，搜索空间过大导致实际应用受限。
*   **直接预测：** 本项目的核心方向。通过学习扰动前后生物网络结构特征的差异，直接输出被扰动的基因集。参考模型包括基于循环因果网络的 **PDGrapher** 和基于因果差分网络的 **CDN**。本项目旨在通过融合大模型特征与“因果启发”的图结构工具，构建更优的分类预测框架。

### 2.2 模型架构与算法实现
模型设计的“第一性原理”将其定义为**多标签分类任务**。关键组件包括：
*   **特征嵌入：** 采用单细胞组学大模型（如CellFM）进行高维特征嵌入，或使用普通嵌入模块刻画全局表达模式。
*   **网络特征学习：** 引入生物调控网络先验（PPI、知识图谱等），通过 **GCN（图卷积网络）**、**GAT（图注意力网络）** 或 **Transformer (Flash Attention)** 学习网络结构与信息传播特性的差异。
*   **输出层：** 利用MLP（多层感知机）输出扰动概率。

### 2.3 实验优化与问题诊断（11.02 - 11.05 日志分析）
在模型调整测试阶段，发现了若干关键技术细节：
*   **正则化行为：** 观察到验证集指标普遍高于训练集。经分析，这是由于训练时开启了 **Dropout** 等正则化技术以防止过拟合，而测试时使用完整模型，从而获得了更佳性能。
*   **Fold Change 计算优化：** 测试发现扰动后靶标基因表达值约一半增加、一半减少。因此，在评估差异表达时，取**Fold Change 的绝对值**进行排序更为合理。
*   **零值处理：** 针对表达谱中存在大量原始值为0的情况，在计算对数Fold Change时引入了 **pseudocount (+1)** 的方案。
*   **环境迁移：** 成功在服务器上配置了 `pdgrapher` 环境，并实现了不依赖大模型的 CellfmEncode 系列脚本（如 `Train_CellfmEncode_GCNqkvGAT_MLP.py`），用于对比基准性能。

**指标定义补充：**
*   **Ranking Score:** 1 - (基因排名 / 总基因数)，越大越好（通常 >0.9）。
*   **NDCG:** 考虑排名位置折损的排序质量，更重视前排正确性。
*   **部分准确预测百分比:** 样本预测前 N 个基因中只要有1个为真实正样本即判定为准确。

## 4. 重要引用与上下文

> **“干预操作会在表型表达中留下‘指纹’，模型通过学习扰动前后转录组表达谱中的生物网络结构特征……直接预测出最可能被扰动的基因集。”**
*   **背景：** 这是课题的核心思想，强调了从“表型指纹”到“靶标预测”的直接关联，避免了间接路径的局限性。

> **“为什么验证集的指标比训练集好？训练集被模型‘看过’并拟合过……原因：使用Dropout等正则化技术的预期行为。”**
*   **背景：** 出自 11.02 的工作日志，解释了深度学习训练中一个看似反直觉但符合逻辑的实验现象。

> **“fold change取绝对值应该比较合理。”**
*   **背景：** 出自 11.02 测试结果分析，基于靶标基因表达值变化趋势（约一半增加一半减少）得出的预处理改进结论。

## 5. 行动建议与应用前景

### 5.1 验证与后续开发建议
1.  **内部基准深化：** 继续在遗传和化学扰动数据集上对比 PDGrapher 和 CDN 等模型，确保模型在复杂调控网络下的稳健性。
2.  **Fold Change 脚本集成：** 在主训练脚本中固化 `pseudocount=1` 及绝对值排序逻辑，以处理基因表达零值问题。
3.  **环境一致性检查：** 确保服务器端 `pdgrapher` 环境与本地测试脚本 `Train_PDG.py` 的指标口径完全一致。

### 5.2 潜在应用场景
*   **药物/化合物机制揭示（MoA）：** 预测未知作用机制化合物的潜在靶点。
*   **个性化医疗：** 输入患者基线与疾病表达谱，输出个性化联合治疗靶点建议。
*   **细胞状态工程：** 预测诱导特定细胞状态变化的扰动靶点，探究复杂疾病机制。
*   **耐药性分析：** 基于响应前后的表达谱对比，预测病人产生获得性或原发性抵抗的作用机制。
